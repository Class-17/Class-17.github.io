Basic Queue

Created2020-11-28|Updated2021-04-06|AlgorithmsZJU_DataStructureLinear_Structure
|Word count:2.1k|Reading time:8min|Post View:

队列的定义

队列(Queue)是一个有序线性表，队列的插入和删除操作时分别在线性表的两个不同端点进行。
设一个队列 $Q = (a_1, a_2, \cdots, a_n)$，那么 $a_1$ 称为队头元素， $a_n$ 称为队尾元素，$a_i$ 排在 $a_{i - 1}$ 的后面 ($1 < i \leq n$)。
如果将元素 $A, B, C, D$ 依次插入队列，第一个从队列中删除的元素将是 $A$ ， 即 先插入的将被率先删除，因此队列通常又被称为 先入先出 (First In First Out, FIFO) 表。

队列的抽象数据类型

  • 类型名称：队列 (Queue)
  • 数据对象集：一个有零个或多个元素的有穷线性表
  • 操作集： 对于一个长度为正整数 $MaxSize$ 的队列 $Q \in Queue$ ，记队列中的任一元素 $X \in ElementType$ ，队列的基本操作主要有：
    1. Queue CreateQueue(int MaxSize): 生成空队列，其最大长度为 MaxSize；
    2. bool IsFull(Queue Q): 判断队列 Q 是否已满。若是返回 true，否则返回 false；
    3. bool IsEmpty(Queue Q): 判断队列 Q 是否为空，若是返回 true， 否则返回 false；
    4. bool Push(Queue Q, ElementType X): 将元素 X 压入队列 Q 中。若队列已满，返回 false，否则将数据元素 X 插入到队列 Q 并返回 true；
    5. ElementType Pop(Queue Q): 删除并返回队列头元素。若队列为空，返回错误信息，否则将队列头元素从队列中删除并返回。

队列的实现

顺序存储实现

采用顺序存储的队列，将 队列头 放在数组下标小的位置，将 队列尾 放在数组下标大的位置，并用两个变量 Front 和 Rear 分别表示队列的头和尾。
当有元素入队时 Rear 向右移动一格（加 $1$），放入队尾元素；当有元素出队，先将 Front 向右移一格（加 $1$）， 再删除队头元素。

假溢出：随着入队、出队的进行，会使整个队列向后移动，这样就会出现队尾指针已经移动到了最后，再有元素入队就会溢出的现象。而事实上此时队列中并未真正“满员”。
考虑到假溢出的问题，选择在队列的顺序存储结构中采用循环队列的方式：
使用公式 “Front % MaxSize” 取余运算实现 折返到起始单元。Rear 和 Front 到达数组端点时，能折返回数组开始处，相当于数组头尾相接。

队列头尾初始化；在使用循环队列时，需要注意 Front 和 Rear 的初始化，以及其所影响的 队列满或空的判断：
Front 和 Rear 初始化为 0， 循环队列中留有一个元素空间不使用，此时 队列满的条件 为：(Rear + 1) % MaxSize == Front，队列为空 时 Front == Rear 。

链式存储实现

采用链式存储的队列，队列头 (Front) 必须指向链表的头节点，队列尾 (Rear) 指向链表的尾节点。
采用链式存储入队和出队的操作实际就是在一个链表的尾部插入节点或者在头部删除节点。

链式队列初始化时，将队列头尾节点初始化为 NULL。
判断 队列是否为空 的条件即为队列头是否指向 NULL。
入队时注意：若入队前队列为空，则队列头尾全部指向该节点；若不为空，则在队列尾节点后接该入队节点，再将队列尾指向该入队节点。

队列的简单应用

Case: 银行业务队列简单模拟

  • PTA 相应 OJ 题链接： 银行业务队列简单模拟

设某银行有A、B两个业务窗口，且处理业务的速度不一样，其中 A 窗口处理速度是 B 窗口的 2 倍，即当 A 窗口每处理完 2 个顾客，B 窗口 处理 1 个顾客。给定到达银行的顾客序列，求业务完成的顺序的顾客序列。
假定不考虑顾客先后到达的时间间隔，并且当窗口处理完 2 个顾客时， A 窗口的顾客优先。

给定一行正整数，其中第一个数字 $N (\leq 1000)$ 为顾客总数，后为 $N$ 位顾客的编号。编号为奇数的顾客需要到 A 窗口办理业务， 为偶数的顾客则要去 B 窗口。

Analysis & Solution

对于 A 和 B 两个业务创建两个队列，分别处理两类业务请求；
根据序列整数的 奇偶性 将各个整数分发到两队列中。

设计针对两个队列处理过程的流程。
设计循环，循环中优先输出 A 队列的顾客，后 判断此次处理的奇偶性，据此判断 B 队列顾客是否输出。
当发现 某一队列为空时，输出另一个队列的所有元素。

小结

队列是一种在一端进行插入而在另一端进行删除的受限的线性表，具有 “先入先出” 的特点，主要操作包括：入队，出队和队空的判断等。

队列的实现可以采用 顺序存储（数组）和 链式存储 两种方式。
顺序存储的实现方式主要采用 循环数组 实现，其中 队空和队满的判断 需要特别注意；
链式存储的实现方式主要采用 单向链表 实现，其中 入队的操作 需要特别注意。

队列的应用非常广， 常见的应用包括：广度优先搜索、操作系统中各种竞争性资源（如 CPU）的管理、实际应用中服务资源的获得（如银行窗口服务）等。
