Heap

|Word count:2.2k|Reading time:8min|Post View:

引子

线性结构队列有一个基本的特征：“先进先出”或者说是“先来先服务”，队列中的元素没有哪个是可以有特权的，前面的元素未处理完，后面的只能等待。
但在现实世界中，许多情况下需要有特权。比如有许多客户在一个打印店排队等待打印资料，某一个客户需要打印数百页的一本书，而排在他后面的客户只需打印两页的一份简历表。我们不妨把前者称为大作业，而后者称为小作业。这时小作业客户对店主和大作业客户提出是否可以先为他打印两页的简历，也就是说时候允许他“插队”。遇到这种情形，店主和大作爷客户往往能接受小作业客户的请求。此时违背了队列先来先服务的策略，是一种称为“小作业优先”的排队策略。“堆”（Heap）正式考虑了适合于特权需求的数据结构，因此堆也通常被称为“优先队列”（Priority Queue）。

堆的定义及表示

  • 堆是特殊的队列，从堆中取出元素是依照元素的优先级大小，而不是元素进入队列的先后顺序。
  • 堆的最常用结构是用二叉树表示，不特指的话为完全二叉树。因为高度为 $h$ 的完全二叉树有节点 $2^{h-1}$ 到 $2^h-1$ 个，且节点排布极其规律，因此，通常不必用指针，而是用数组来实现堆的存储。
  • 队中的元素按照完全二叉树的层序存储，根节点存放再数组的起始处，接着是其子节点，一层层下去，直到最后一个节点。注：数组起始单元为1。这样可以使得很容易从子节点找到其父节点。对于下标为 i 的节点，其父节点的下标为 ⌊i / 2⌋。反过来，找节点 i 的左、右子节点也十分方便，分别为 2i 和 2i + 1。
  • 在最大堆中任一结点的值大于等于其子节点的值；在最小堆中，任一结点的值小于等于其子节点的值；兄弟节点直接不存在约束关系。

堆的抽象数据类型（以最小堆为例）

  • 类型名称: 最小堆（MinHeap）
  • 数据对象集： 一个有 $N > 0 $个元素的最大堆 $H$ 是一颗完全二叉树，每个节点上的元素值不大于其子节点的元素值。
  • 操作集：对于任意最多有 $MaxSize$个元素的最小堆 $H \in MinHeap$ ，元素 $X \in ElementType$ ，主要有以下操作：
    1. MinHeap CreateHeap(int MaxSize): 创建空的最小堆，其最大长度为 MaxSize；
    2. bool IsFull(MinHeap H): 判断最小堆 H 是否已满，若是返回 true；否则返回 false；
    3. bool Insert(MinHeap H, ElementType X): 将元素 X 插入最小堆 H。若堆已满，返回 false ；否则将数据元素 X 插入到堆 H 并返回 true；
    4. bool IsEmpty(MinHeap H): 判断最小堆 H 是否为空，若是返回 true；否则返回 false；
    5. ElementType Delete(MinHeap H): 删除并返回 H 中最小元素。

堆的实现（以最小堆为例）

最小堆的创建

根据用户输入的 MaxSize 建立空的最小堆时，数组应该有 MaxSize + 1 个元素，因为数组起始单元为 1，元素是存储在第 1 ~ MaxSize 个单元中的。 通常第 0 个单元是无用的，但是如果事先知道知道堆中所有元素的取值范围，也可以给第 0 个单元赋一个特殊的值 MINDATA ，这个值比堆中任何一个可能的元素都要大。

最小堆的插入

  • 最小堆中插入一个新元素以后，新增节点既要保证最大堆仍是完全二叉树，节点之间元素值的大小也要满足最小堆的性质，因此，一般情况下要移动元素。是否移动、多少元素要移动，取决于要插入元素以及最小堆中已有元素数值的大小。
  • 完成一个元素的最小堆插入操作，需从完全二叉树的新增节点开始，顺着其父节点到根节点的路径，将路径上各节点一次与新元素值进行比较，当一节点的值大于新元素的值，就下移这个节点的元素，直到有节点的值小于新元素的值或根节点也下移为止，空出的节点位置就是新元素插入点。
  • 最小堆的插入：从新增的最后一个节点的父节点开始，用要插入的元素向下过滤上层节点。

最小堆的删除

  • 最小堆的删除实际上是取出根节点的最小值元素，同时删除一个堆的节点。与插入操作类似，删除操作后，最小堆任然要是完全二叉树，节点元素值的大小仍然要满足最大堆的性质。因此，删除的节点应该是数组元素的最后一个单元。那么取走根节点的元素后，对的最后一个节点必须重新放置。
  • 将删除节点（数组中最后一个元素单元）的元素作为假设的根节点，依次与其下层的子节点进行比较，如果大于子节点的元素值，从两个子节点中选择值小的元素上移一层直到在某一点上，比较结果是小于两个子节点的值，此时空的节点就是要放置删除节点元素的正确位置。
  • 最小堆的删除：从根节点开始，用最小堆中的最后一个元素向上过滤下层节点。

最小堆的组建

当已有 N 个元素按照最小堆的要求，一个一个进行插入操作，其时间代价最大为 O(N logN)。这里选择一种更加简便高效的方式。

1.将 N 个元素按输入顺序存入二叉树中，这一步要求满足完全二叉树的结构特性，而不管其有序性。
2. 调整各个节点元素，满足最大堆的有序性。
