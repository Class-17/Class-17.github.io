AOJ 2021下学年第15周周赛题解

比赛链接： https://www.webturing.com/contest.php?cid=1674

A. 数字分类

标签： 数学

将读入的数字分别放到两个数组内，输入结束后，依次输出即可。

B. 约分最简分式

标签： 数学

读入后，求解 a 和 b 的最大公约数，输出 a 和 b 分别除以最大公约数的结果即可。

C. 最小矩形

标签： 数学

由题目条件，可简单得出以下结论：

  • 矩形的长为横坐标最大的点和横坐标最小的点的横坐标之差。
  • 矩形的宽为纵坐标最大的点和纵坐标最小的点的纵坐标之差。

故而遍历所有点后，可得到最小矩形的长和宽，相乘输出即可。

D. 求矩形重叠的面积

标签：数学

两矩形重叠部分为一个新的矩形，可思考求解重叠部分矩形的长和宽。

我们把矩形的长和宽分别抽象到一维上。

对于以上情况通过观察可发现：

  • 矩形重叠部分的长，也就是线段 $ab$ 和线段 $cd$ 重叠的长度，故重叠部分矩形的长为：$ab + cd - ad$

  • 同样的，矩形重叠部分的宽，即为线段 $aj$ 和 $cl$ 重合的部分。故重叠部分矩形的宽为：$aj + cl - al$

由上述情况，我们可推导出适用于所有矩形重叠情况的计算方法。

$ab + cd = l_a + l_b$
$ad = max(x_a + l_a, x_b + l_b) - min(x_a, x_b)$
$aj + cl = w_a + w_b$
$al = max(y_a, y_b) - min(y_a - w_a, y_b - w_b)$

计算出的矩形长或宽存在小于零的情况，则说明两矩形不重叠。

E. 小圆的面积

标签: 数学

本题我们根据圆的相关定理，可以建立小圆半径和边长之间的关系式：

$$
r^2 + \frac{d^2}{4} = (d - r)^2
$$
$$
r^2 + \frac{d^2}{4} = d^2 - 2dr + r^2
$$
$$
2dr =\frac{3}{4}d^2
$$
$$
r = \frac{3}{8}d
$$

所以最后的答案就是：

$$
area = \frac{9}{64}d\pi^2
$$

F. 数字黑洞

标签： 数学

本题根据题意模拟求解即可

G. 钢琴块

标签： 排序；贪心

首先将所有钢琴块的到来时间进行排序，然后依次处理：

  1. 定义一个数组 int finger[4] ，初始状态为 0 ，表示 $4$ 根手指可用的时间。
  2. 遇到一个 tile 我们就去寻找 finger 里第一个小于等于 tile 的数，然后将其值更新为 tile + d ，如果没有 finger 满足条件就说明无法完成，返回 -1。
  3. 以上操作可以使得在反应速度足够的情况下，我们只会使用到前 $i (1 \le i \le 4)$ 个数，这样，我们维护一个值 max_num 便可以知道，每次使用的 finger 的最大编号，这样在可以完成游戏的情况下，便得到了答案。

时间复杂度： $O(NlogN)$

H. 数字拼接

标签： 贪心； 排序；

对于两个数字 $a$ 和 $b$ ，我们可以讨论以下问题：

  • $a$ 和 $b$ 谁在前可以使得拼接结果最大。

此时我们可以思考 $a + b$ 与 $b + a$ 的大小关系，若 $a + b$ > $b + a$ 说明 $a$ 应该放在 $b$ 前，此时我们就分析出了 $nums$ 数组中元素的大小关系，可写出 comp 函数来对 $nums$ 数组排序，之后将 $nums$ 数组拼接起来，便是答案。

时间复杂度： $O(NlogN)$

I. 字符串计数

标签： 字符串； 动态规划；

遍历字符串 $s$ , 当遇到 abadedde 里出现的字符时，对于 a, b, d, e 我们做不同的处理。
我们使用数组统计以下字符串出现的数量:

dp[0] -> "a"
dp[1] -> "ab"
dp[2] -> "aba"
dp[3] -> "abad"
dp[4] -> "abade"
dp[5] -> "abaded"
dp[6] -> "abadedd"
dp[7] -> "abadedde"

下面对当前字符遇到的 a, b, c, d 四种情况进行分析：

  • s[i] == 'a' ，此时 "a" 的数量加一，此外若存在字符串 "ab" 那么 "aba" 的数量就会加上 "ab" 字符串当前的数量。

  • s[i] == 'b' ， 此时 "ab" 的数量会加上 "a" 的数量。

  • s[i] == 'd' ，此时 "abad" 的数量会加上 "aba" 的数量， "abaded" 的数量会加上 "abadedd" 的数量会加上 "abaded" 的数量；注意如果以滚动数组实现，要从大到小更新 dp 数组。

  • s[i] == 'e' , 此时 "abade" 的数量会加上 "abad" 的数量， "abadedde 的数量会加上 "abadedd" 的数量。

时间复杂度： $O(N)$

J. 01 方阵

标签： 状态压缩；

由题目数据我们可以发现， $N$ 的值明显很小，而且对于最终的答案，我们的结果应该是由对于 $i$ 行 或 $j$ 行翻转或否的问题。

如果我们拆分 $N$ 行的答案，我们会得到 $2^{N}$ 种结果， 同样的拆分 $M$ 列的答案，我们会得到 $2^{M}$ 种结果。

但是我们可以发现，当对一个行或者列的答案进行枚举的时候，另外一维的元素，我们翻转或不反转取两者中的最大值即可

例如

0 1 0 1 0
1 0 0 0 1

此时我们对于第一行选择翻转（1表示），第二行不翻转（0表示），
则此种翻转方式对于第一列的结果是 {1, 1} ，而此时我们讨论对第一列翻转还是不反转，显然就是取 $max(sum_0, 2 - sum_1)$ 如果 $0$ 比较多的话，就不翻转，反之我们翻转该列。

那么对于行的翻转结果，共 $2^{N}$ 种结果，也就是 $2^N$ 种 $0$ 和 $1$ 的组合，我们可以用一个 int 类型数据上的 二进制 位上的 $0$ 和 $1$ 来表示这种状态。

对于每列的初始值，我们同样选择用一个 int 类型数据上的 二进制 位上的 $0$ 和 $1$ 来进行表示，这样，我们在计算 当前 行 状态对于 当前 列 的影响时，便可以使用二进制运算: 异或 ，来快速得到结果。

最后我们使用位运算得到 当前列状态 的结果中 二进制位上 $1$ 的个数，这样我们便取得了，在 当前行选择状态 下，该列能得到的最大结果，之后对于这种 行选择状态 ， 我们将 $M$ 列全部枚举一遍，即可得到这种 行选择状态 下能得到的最大结果。

于是将每种 行选择状态 使用上述方法枚举一遍，便可得到最优解。

时间复杂度: $O(2^N \cdot M)$
