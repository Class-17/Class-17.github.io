数据结构（英语：data structure）是计算机中存储、组织数据的方式。

维基百科数据结构

引例: 一元多项式的求值问题

课题

一元多项式的标准表达式可以写为： $ f(x) = a_0 + a_1x +\cdots + a_{n-1}x^{n-1} + a_nx^n $ 。 现给定一个多项式的阶数 n ， 并将全体系数 $ \{a_i\}_{i=0}^n $ 存放在数组 a[] 里，编写程序计算多项式在给定点 x 处的值。

分析

直接法： 根据题意最直接的办法是根据多项式的标准表达式 $ f(x) = \sum_{i=0}^n a_ix^i $ 通过循环累计求和来实现计算。

代码实现：计算多项式函数值的直接法

秦九韶法： 不断提取公因式 x 来减少乘法的运算次数，把多项式写为：
$$ f(x) = a_0 + x(a_1 + x(\cdots(a_{n-1} + x(a_n))\cdots)) $$

代码实现：计算多项式函数值的秦九韶法

由上述两种求解表达式的方式，我可以观察到两种方式代码实现的长度基本相同，但是实际上秦九韶法要比直接法快很多。
接下来介绍 C/C++ 中的程序运行时长计算工具。

Algorithm RunTime Test Tools

  • C/C++ 中提供了 clock() 工具，帮助计算程序或函数运行时间。

使用方法

  1. 调用头文件 time.h。
  2. 调用函数 clock() 获取时钟打点数。
  3. 使用常数 CLK_TCK 或 CLOCKS_PER_SEC 计算运行时，单位为秒。
  4. 毫秒单位输出待测试函数运行时。
  5. 由于有时出现测试代码运行时非常短（小于一次打点数），采用重复调用函数的方式测得实际运行时：设置常数 REPEAT，最后运行时结果除以该常数，从而得到运行时较短的测试代码的运行时。

“直接法” 与 “秦九韶法” 的比较

给定一元多项式 $ f(x) = \sum_{i=0}^n i \cdot x^i $ 分别使用 直接法 和 秦九韶法 求 $f(1.1)$ 在 $n$ 为 1e4, 1e5, 1e6, 1e7, 1e8 的数量级时的效率差别。

效率比较

直接法 秦九韶法
n = 999 1.27ms 0.07ms
n = 9999 13.52ms 0.69ms
n = 99999 145.99ms 6.97ms
n = 999999 1513.58ms 70.24ms
n = 9999999 15720.16ms 698.89ms

由表可观察到，秦九韶法比直接法快了一个数量级。
秦九韶算法更胜一筹！ 下面对两种算法进行详细分析。

  • 直接算法：

    代码执行了 $n$ 次语句 p = a[i] * pow(x, i);
    每次涉及 $i$ 次乘法和 $1$ 次加法运算，于是全部计算设计 $n$ 次加法和
    $(1 + 2 + \cdots + n) = (n^2 + n) / 2$ 次乘法。

  • 秦九韶算法：

    代码执行了 $n$ 次语句 p = a[i] + x * p;
    全部计算涉及 $n$ 次加法和 $n$ 次乘法。

算法分析

算法简介

“算法”（Algorithm）一词是由 Algorism 衍生而来，而 Algorism 源自一本波斯数学教材，原意为“算术”。
算法设计是一门艺术。解决同一个问题，一般有多种算法，但漂亮的算法与其他算法相比往往由天壤之别。
一般而言，算法是一个有限指令集，它接受一些输入（有些情况下不需要输入），产生输出，并一定在有限步骤之后终止。算法的每一条指令必须有充分明确的目标，不可以有歧义；必须在计算机能够处理的范围之内；且其描述应不依赖于任何一种计算机语言及其具体的实现手段。

算法复杂度

除了算法的描述风格之外，具体衡量、比较算法优劣的主要指标主要有以下两个：

  1. 空间复杂度 $S(n)$ —— 根据算法写成的程序在执行时占用储存单元的长度。这个长度往往与输入数据的规模 $n$ 有关。空间复杂度过高的算法可能导致使用的内存超限，造成程序非正常中断。

  2. 时间复杂度 $T(n)$ —— 根据算法写成的程序在执行时耗费时间的长度。这个长度往往也与输入数据的规模 $n$ 有关。时间复杂度过高的低效算法可能导致我们在有生之年都等不到运行结果。

“直接算法”与“秦九韶算法”的算法对比

由以上两种算法的分析比较可以发现：

简单直接算法时间复杂度为：$T_1(n) = C_1n^2 + C_2n$ ,其中 $n$ 为输入多项式的阶数；$C_1$ 为执行 $1/2$ 次乘法需要的时间，$C_2$ 是执行 $1$ 次加法和 $1/2$次乘法需要的时间。

秦九韶算法时间复杂度为：$T_2(n) = C \cdot n$，其中 $n$ 为输入多项式的阶数；$C$ 是执行 $1$ 次加法和乘法需要的时间。

由此可得，对于充分大的 $n$， $T_2(n)$ 总会比 $T_1(n)$ 小，即秦九韶算法比简单直接算法快，而且 $n$ 越大，快的越明显。
