堆栈的定义

堆栈（Stack）可以认为是 具有一定约束的线性表，插入和删除的操作都作用在一个称为栈顶（Top）的端点位置。
在堆栈中，通常把数据插入称为 压入栈（Push），而数据的删除可看作从堆栈中取出数据，叫做 弹出栈（Pop）。
由于这种特性，最后入栈的数据将被最先弹出，所以堆栈也被称为 后入先出 (Last In First Out, LIFO) 表。

堆栈的抽象数据类型

  • 类型名称：堆栈（Stack）
  • 数据对象集：一个有零个或多个元素的有穷线性表
  • 操作集：对于一个具体的长度为正整数 $MaxSize$ 的堆栈 $S \in Stack$，记堆栈中的任一元素 $X \in ElementType$，堆栈的基本操作有：
    1. Stack CreateStack(int MaxSize): 生成空堆栈，其最大长度为 MaxSize；
    2. bool IsFull(Stack S): 判断堆栈 S 是否已满。若是返回 true，否则返回 false；
    3. bool IsEmpty(Stack S): 判断堆栈 S 是否为空，若是返回 true， 否则返回 false；
    4. bool Push(Stack S, ElementType X): 将元素 X 压入堆栈 S 中。若堆栈已满，返回 false，否则将数据元素 X 插入到堆栈 S 栈顶处并返回 true；
    5. ElementType Pop(Stack S): 删除并返回栈顶元素。若堆栈为空，返回错误信息，否则将栈顶元素从堆栈中删除并返回。

堆栈的实现

由于堆栈是线性表，因而堆栈的存储结构可采用 顺序 和 链式 两种形式。顺序存储的堆栈称为 顺序栈，链式存储的堆栈称为 链栈。

顺序存储实现

栈的顺序存储结构通常由一个 一维数组 和一个 记录栈顶元素位置的变量 组成，另外，可以使用一个变量来存储堆栈的最大容量 MaxSize， 这样方便判断什么时候堆栈是满的。
使用一维数组 Data[MaxSize]（下标 0 ~ MaxSize - 1）存储一个栈的元素。
将栈底放在数组下标小的一端，栈顶位置使用整型变量 Top 记录当前栈顶元素的下标值。当 Top = -1时，表示空栈；当 Top = MaxSize - 1 时，表示满栈。

  • 顺序栈 C 语言实现

链式存储实现

栈的链式存储结构（链栈）与单链表类似，但其操作受限制， 插入和删除操作只能在链栈的栈顶进行。为了简便算法，链栈可以带一空的表头结点，表头结点后面的第一个节点就是链栈的栈顶结点，栈中的其他节点通过它们的指针 Next 链接起来，栈底节点的 Next 为NULL。

链栈不再包含 Top 和 MaxSize 变量，其操作集不再包含 bool IsFull(Stack S) ,使用也更加灵活。

堆栈的简单应用

Case: 汉诺塔非递归实现

  • PTA 相应 OJ 题链接： 汉诺塔的非递归实现

借助堆栈以非递归（循环）的方式求解汉诺塔问题 (n, a, c, b) ，即将 n 个盘子从 起始柱（标记为 a ）通过 借助柱（标记为 b ）移动到 目标柱（标记为 c），并保证每个移动都符合汉诺塔问题的要求。

Analysis

汉诺塔问题可抽象为：将 n 个圆盘从 “起始柱” 移动到 “目标柱”。
继续抽象原问题为：将 n - 1 个圆盘从 “起始柱” 移动到 “借助柱”，再将第 n 个圆盘从 “起始柱” 移动到 “目标柱”，最后将 n - 1 个圆盘从 “借助柱” 移动到 “目标柱”。
故汉诺塔求解问题的基本思路为：不断将 n 个圆盘的汉诺塔问题转化为 2 个 n - 1 圆盘的汉诺塔问题。

Solution

设 n 盘问题为 (n, a, c, b)， 则问题的求解可 转化 为对
(n - 1, a, b, c), (1, a, c, b), (n - 1, b, c, a)
这三个问题的求解。

故可建立堆栈，将待求解问题放入堆栈，然后不断将栈顶的问题分解，在将分解出的 n > 1 的新问题放入堆栈，如此不断循环直到堆栈为空，问题求解结束。
将分解出的上述三个问题压入堆栈时，应该按照 “需要先求解的问题后压入” 的顺序，也就是压入顺序为:
(n - 1, b, c, a) -> (1, a, b, c) -> (n - 1, a, b, c)

小结

堆栈是一种只在一端做插入删除的受限的线性表，具有 “后入先出” 的特点，主要操作包括：入栈、出栈和栈空的判断等。

堆栈的实现可以采用 顺序存储（数组）和 链式存储 两种方式。

堆栈的应用非常广，常见的应用包括：表达式求值、函数调用、递归实现、深度优先搜索等。
